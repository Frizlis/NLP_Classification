# News class prediction
Описание проекта: Проект нацелен на классификацию новостных данных по 42 классам, для дальнейшей оценки их значимсти.

Датасет для обучения состоит из 210 тыс. новостей ХаффПост с 2012 по 2022 год. 

Проект включает в себя следующие этапы:

  * Проверка на корректность и качество данных (файл 01_eda.ipynb)
  * Детальное изучение признаков и таргета (файл 01_eda.ipynb)
  * Построение базовой модели и подсчет метрик (файл 02_baseline.ipynb)
  * Обучение основных моделей и тестирование лучших (файл 03_nlp.ipynb)
    
По итогу EDA было определено, что в датасете отсутствуют пропуски, но имеется около 13 дубликатов.

Имеется существенный дисбаланс среди таргетов. Классы POLITICS, WELLNESS, ENTERTAIMENT шире представлены чем остальные. Лишь 14 классов из 42 поредсавлены более чем на 2%.

Основная текстовая информация содержится в колонке "short_description". Расперделение длинн текстов в этой колонке выглядит нестандартно и имеет несколько крупных пиков. Максимальная длинна текста в этой колонке составляет 243 слова в классе WORLDPOST. А минимальная 0, этот минимум встречается в 37 картегориях. Пустые значения распределены по классам не равномерно.

Нет новостей в которых все 3 основные столбца пустые. Если "short_description" и "headline" пусты то "authors" имеет запись.

Без фильтрации превалируют общие слова и предлоги по котором сложно идентифицировать к какому классу текстов оно пренадлежит.

Дополнительная текстовая информация содержится в колонке "headline". Распределение длинн "headline" имеет более стандартное распределение. Среди наиболее популярных слов встречаются те что более ярки по контексту. Также встречается много чисел.

В колоонке "authors" встречается более 29169 уникальных записей.

При построрении Baseline был выбран подход с использованием CountVectorizer, TfidfVectorizer и классических ML методов.
По полученным на крос валидации методам лучшим средим базовых моделей оказалась связка CountVectorizer и LogisticRegression.
